# Security Policy

This is a sanitized evaluation repository. It must never contain production credentials, customer data, database exports, production source snapshots, or privileged operational details.

If sensitive information is discovered:

1. Stop sharing the repository.
2. Remove access for external collaborators.
3. Determine whether the material exists in Git history.
4. Rotate any exposed credential immediately.
5. Remove the material from current content and history as required.
6. Review related systems and access logs.

Security concerns should be reported privately to the repository owner rather than filed as a public issue.
